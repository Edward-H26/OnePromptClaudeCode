# Retro

Use this skill for retrospectives and process review.

## Repo Rules

- Base the retrospective on repo-local facts, recent diffs, and available session artifacts.
- Do not assume `~/.gstack` metrics or global history exist.
- Keep the output useful for the current repo and team workflow.

## Workflow

1. Summarize what changed and what worked.
2. Identify friction, failures, and repeatable wins.
3. Recommend a small number of concrete process improvements.
