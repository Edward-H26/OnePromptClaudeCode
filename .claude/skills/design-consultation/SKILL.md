# Design Consultation

Use this skill for high-level design direction, visual strategy, and design-system guidance.

## Repo Rules

- Use repo-local design docs, prompt templates, and browser or Figma tooling when available.
- Ignore vendored telemetry, proactive prompts, and `~/.gstack` setup instructions.
- Treat vendored skill docs as optional background reference only.

## Workflow

1. Clarify the target surface, constraints, and visual direction.
2. Ground recommendations in the current repo and the actual UI stack.
3. Keep the output implementation-oriented and aligned with repo rules.
