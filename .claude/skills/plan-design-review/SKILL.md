# Plan Design Review

Use this skill to review design quality before implementation or before major UI changes.

## Repo Rules

- Keep the review focused on the current repo and planned UI outcome.
- Ignore vendored telemetry and global-home setup.
- Do not convert this into an implementation workflow unless the user asks.

## Workflow

1. Evaluate hierarchy, clarity, responsiveness, and distinctiveness.
2. Flag likely weak spots before code is written.
3. Return concrete design guidance and open questions.
